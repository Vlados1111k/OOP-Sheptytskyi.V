# План ітерації №2 (Lab 35)

### 1. Сценарії з Backlog в реалізацію:
- Додавання фільму через консоль.
- Збереження та завантаження бази фільмів у JSON.
- Пошук фільмів за жанром та тривалістю (LINQ).
- Розрахунок вартості квитка з урахуванням знижок (Strategy Pattern).

### 2. Точки розширення:
- Впровадження `IRepository` для абстракції роботи з даними.
- Патерн **Strategy** для системи знижок (діти, студенти, VIP).

### 3. Ризики:
- Складність реалізації асинхронного завантаження при старті.
- Потенційні конфлікти при читанні пошкодженого JSON-файлу.
