# Lab 7: Реалізація патерну Retry

**Виконав:** Студент групи [Ваша група], Шептицький В. (Варіант 17)

## Мета
Розробка механізму стійкості до помилок (Fault Tolerance) при роботі з нестабільними ресурсами.

## Реалізація
1. **RetryHelper**: Універсальний клас для повторного виконання операцій з експоненційною затримкою.
2. **FileProcessor**: Імітація читання файлу з обробкою `IOException`.
3. **NetworkClient**: Імітація HTTP-запитів з обробкою `HttpRequestException`.

## Результат
Програма демонструє успішне відновлення роботи після серії тимчасових збоїв за допомогою конфігурованої стратегії Retry.
