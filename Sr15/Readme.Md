## 2. Принцип єдиної відповідальності (SRP)

### 2.1. Дотримання (High Cohesion)
- **Клас:** `JsonTextReader`
- **Відповідальність:** Виключно лексичний аналіз та токенізація JSON-потоку.
- **Обґрунтування:** Модуль ізольований від бізнес-логіки об'єктів, що забезпечує низьку зв'язність та спрощує тестування парсингу.

- **Клас:** `DefaultContractResolver`
- **Відповідальність:** Дефініція правил відображення CLR-типів.
- **Обґрунтування:** Відокремлює побудову метаданих від процесів серіалізації.

### 2.2. Порушення (God Object)
- **Клас:** `JsonSerializer`
- **Проблема:** Агрегує конфігурацію, стратегії кешування та координацію потоків даних.
- **Наслідки:** Висока когнітивна складність та ризики регресії при модифікації суміжних аспектів функціонування.

---

## 3. Принцип відкритості/закритості (OCP)

### 3.1. Дотримання (Extensibility)
- **Механізм:** Абстракція `JsonConverter` (Патерн Strategy).
- **Обґрунтування:** Архітектура відкрита для підтримки кастомних типів даних через успадкування, без інвазивного втручання в інкапсульовану логіку ядра бібліотеки.

```csharp
public abstract class JsonConverter {
    public abstract void WriteJson(JsonWriter writer, object value, JsonSerializer serializer);
    public abstract object ReadJson(JsonReader reader, Type objectType, object existingValue, JsonSerializer serializer);
    }

Проєкт демонструє зрілу архітектуру. Високий рівень розширюваності через OCP нівелює певні порушення SRP у центральних компонентах, забезпечуючи життєздатність системи в довгостроковій перспективі.
